# Pragmatic Programming

## Overview

This skill provides a comprehensive reference and guide to Pragmatic Programming concepts. It covers a wide range of topics from basic code design principles to complex architectural patterns.

## When to Use This Skill

Use this skill when you need to:
- Understand specific software engineering concepts (e.g., "What is Design by Contract?").
- Get advice on architectural decisions (e.g., "Should I use Microservices?").
- Improve code quality through refactoring and design principles (e.g., "How do I apply DRY?").
- Explore design patterns and their trade-offs.

## Concepts Library

The core knowledge base of this skill is located in the `references/` directory.

### [Concepts Reference](references/concepts.md)

This document includes detailed explanations of:

- **Code Design**: Design by Contract, Tracer Bullets, DRY, Refactoring, Law of Demeter, Property-Based Testing, Mixins, TDD, Multiple Inheritance, Crash Early, Technical Debt, Class Invariant, Prototyping.
- **Complexity**: Big-O Notation.
- **Architecture**: Actor Model, Orthogonality, Microservices, Uniform Access Principle, ETC.
- **Design Patterns**: Observer, Blackboards, Visitor, Composite, Singleton, Reactive Programming, "Tell, Don't Ask".
